# iOS geolocation permission — Design (quick spec)

**Status:** Approved (quick spec)
**Date:** 2026-06-10
**Scope:** Bug fix found while testing the app in Xcode. Standalone — one branch / one PR.

## Problem

On the native iOS app the map never asks for location permission, so it never
centres on the user. The app calls **web `navigator.geolocation.getCurrentPosition`**
in [src/components/MapView.tsx](../../../src/components/MapView.tsx) (one effect,
~line 343). Capacitor's iOS webview (WKWebView) does **not** bridge HTML5
`navigator.geolocation` to CoreLocation, so the call silently fails and no
permission prompt is shown. `Info.plist` already has
`NSLocationWhenInUseUsageDescription`, and `@capacitor/geolocation` is **not**
installed.

## Goal

On iOS, the app requests "When In Use" location permission on first map load and
centres on the user when granted; on web it keeps working as before. Same
auto-centre behaviour, same silent fallback to the default centre on
denial/error.

## Approach

Use the official **`@capacitor/geolocation`** plugin, which bridges to native
CoreLocation on iOS (triggering the prompt) and wraps `navigator.geolocation` on
web — one code path for both.

1. **Add dependency:** `@capacitor/geolocation@^8` (match Capacitor core `^8.3.4`).
2. **`src/components/MapView.tsx`:** replace the single `navigator.geolocation`
   effect with the plugin's Promise API — `Geolocation.requestPermissions()`
   (best-effort, wrapped so a web hiccup doesn't block) then
   `Geolocation.getCurrentPosition({ enableHighAccuracy: false, timeout: 20000,
   maximumAge: 120000 })`. Keep the exact success behaviour (`setMapCenter`,
   `setZoom`, `pendingGeoCenterRef`, `tryFlyToPendingGeoCenter`) and the silent
   error fallback. Add a `cancelled` guard for unmount.
3. **`npx cap sync ios`** so the native project registers the plugin.

## Decisions

- One unified code path (the plugin works on web too) — do **not** keep a
  separate `navigator.geolocation` branch.
- `requestPermissions()` is best-effort and wrapped in its own try/catch so a web
  Permissions-API quirk can't block `getCurrentPosition` (which also prompts).
- No new UI, no opt-in toggle — preserve the current "auto-centre on load"
  behaviour (YAGNI).

## Out of scope

- Background location, watch/continuous tracking, an in-app "locate me" button.
- Android (iOS-only project today).

## Files touched

- Modify: `package.json` / `package-lock.json` (add `@capacitor/geolocation`)
- Modify: `src/components/MapView.tsx` (one effect + one import)
- Possibly: `ios/App/App/capacitor.config.json` and Pods, regenerated by
  `npx cap sync ios` (committed if changed)

## Verification

- `npm run build`, `npm run lint`, `npm test` (15/15) all pass.
- **iOS (user, Xcode):** run the app, confirm the system **"Allow … to use your
  location"** prompt appears on first map load, and on Allow the map centres on
  the user. Confirm Deny keeps the default centre with no crash.
- Web: `npm run dev`, confirm the browser still prompts and centres as before.

## Risks

- Plugin/Capacitor version mismatch → pin to `^8`.
- Native verification needs Xcode (done by the user); web + build/lint/test are
  verifiable here.
